Introduction

HTML (HyperText Markup Language) is a markup language used to define the structure of a webpage. It is the skeleton of every website you will ever see. HTML tells the browser how to display content such as text, images, links, and headings.

It forms the foundation of web development, working together with CSS for styling and JavaScript for interactivity.

Why HTML

  • Foundation of the web

    • Every website you visit is built using HTML. Without it, web pages cannot exist.

  • Defines page structure

    • It organizes content like text, images, links, buttons, and forms so browsers know how to display them.

  • Easy to learn

    • HTML is beginner-friendly and has simple rules, making it a great starting point for web development.

  • Works with CSS & JavaScript

    • HTML gives structure, CSS adds design, and JavaScript adds interactivity — together they create modern websites.

  • Universal and essential

    • HTML is supported by all browsers and devices, making it the universal language of the web.

Basic Syntax

HTML is written using tags that tell the browser how to display content. Most tags come in pairs, with an opening tag and a closing tag.

It uses an opening tag like <tagname> and a closing tag like </tagname>.

HTML Hierarchy

HTML follows a parent–child structure, where elements can contain other elements.

At the top level, you have the <html> element, and inside it you place the <head> and <body> sections. Inside the <body>, you place the content elements like headings, paragraphs, lists, images, and more.

In simple terms: HTML is nested — big elements hold smaller elements.

This organization creates a clear hierarchy that the browser uses to understand how content is related and how it should be displayed.

Semantic Tags

Semantic tags in HTML are elements that clearly describe their meaning and purpose to both the browser and developers. Instead of simply creating structure, they tell what the content represents.

Examples include <header>, <nav>, <main>, <section>, <article>, and <footer>. These tags improve code readability, accessibility, and search engine understanding, making web pages more organized and meaningful compared to generic containers like <div>.
